Avoid Hardcoding Social Security Numbers (SSNs)

This rule checks for hardcoded Social Security Numbers (SSNs) in Python code.

Non-Compliant Example

        
# Non-compliant code example
ssn_person1 = "123-45-6789"
ssn_person2 = "987-65-4321"
def example_function():
# Using the SSN inside a function
ssn_in_function = "111-22-3333"
return ssn_in_function

Compliant Example

        
# Compliant code example
import os
ssn_person1 = os.getenv('SSN_PERSON1')
ssn_person2 = os.getenv('SSN_PERSON2')

def example_function():
# Using the SSN inside a function
ssn_in_function = os.getenv('SSN_IN_FUNCTION')
return ssn_in_function

Rationale

Hardcoding Social Security Numbers (SSNs) in source code poses a significant security risk. If the source code is leaked or shared, the hardcoded SSNs can be misused for identity theft or other malicious activities. Using environment variables to store and retrieve sensitive information helps mitigate this risk.

How to Fix Violations

To fix the violation, move the sensitive SSNs to environment variables and access them using os.getenv or a similar method appropriate for your programming environment. This approach ensures that sensitive information is not exposed in the source code.
