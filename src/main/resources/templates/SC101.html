No Hardcoded API Keys

This rule discourages the use of hardcoded API keys within the code to prevent security risks.

Non-Compliant Example

        
# Non-compliant code example
API_KEY = "12345-abcde-67890-fghij"
response = requests.get(f"https://api.example.com/data?apikey={API_KEY}")

Compliant Example

        
# Compliant code example
import os
API_KEY = os.getenv("API_KEY")
response = requests.get(f"https://api.example.com/data?apikey={API_KEY}")

Rationale

Hardcoding API keys within the code can lead to security vulnerabilities if the code is exposed. Storing API keys in environment variables or configuration files ensures that sensitive information is kept secure and can be easily changed without modifying the code.
