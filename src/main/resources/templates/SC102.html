Avoid Hardcoding AWS Access Keys

This rule checks for hardcoded AWS access keys (starting with AKIA or ASIA) in Python code.

Non-Compliant Example

        
# Non-compliant code example
aws_key_akia = "AKIAEXAMPLE1234567890"
aws_key_asia = "ASIAEXAMPLE1234567890"

def example_function():
    # Using the AWS key inside a function
    aws_key_in_function = "AKIAANOTHEREXAMPLE123456"
    return aws_key_in_function

Compliant Example

        
# Compliant code example
import os

aws_key_akia = os.getenv('AWS_ACCESS_KEY_ID')
aws_key_asia = os.getenv('AWS_SECRET_ACCESS_KEY')

def example_function():
    # Using the AWS key inside a function
    aws_key_in_function = os.getenv('ANOTHER_AWS_ACCESS_KEY_ID')
    return aws_key_in_function

Rationale

Hardcoding AWS access keys in the source code is a security risk. If the source code is leaked or shared, the hardcoded keys can be used to gain unauthorized access to AWS resources. Using environment variables to store and retrieve sensitive information helps in mitigating this risk.

How to Fix Violations

To fix the violation, move the sensitive keys to environment variables and access them using os.getenv or a similar method appropriate for your programming environment. This approach ensures that sensitive information is not exposed in the source code.
